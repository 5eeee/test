# Настройка GitHub Pages для Studio <4:30/> Mini App

## Обзор

Этот файл содержит пошаговые инструкции по развертыванию вашего Telegram Mini App на GitHub Pages.

## Подготовка репозитория

1. Создайте новый репозиторий на GitHub или используйте существующий
2. Загрузите все файлы из текущего проекта в репозиторий
3. Убедитесь, что структура файлов сохранена, особенно папка `studio-mini-app`

## Настройка GitHub Actions

Файл конфигурации GitHub Actions уже создан в `.github/workflows/deploy.yml`. Он автоматически:

1. Установит все зависимости Node.js
2. Соберет приложение с помощью Vite
3. Разместит результат сборки на GitHub Pages

## Шаги для активации GitHub Pages

1. Перейдите в свой репозиторий на GitHub
2. Перейдите в раздел "Settings"
3. Найдите раздел "Pages" в левой боковой панели
4. В разделе "Source" выберите "GitHub Actions"
5. Убедитесь, что ветка указана как "main" или "master"

## Проверка развертывания

1. Перейдите в раздел "Actions" в вашем репозитории
2. Вы должны увидеть запущенный workflow "Deploy to GitHub Pages"
3. Дождитесь завершения процесса сборки (это может занять 2-5 минут)
4. После успешной сборки сайт будет доступен по адресу:
   `https://<ваш-username>.github.io/<название-репозитория>`

## Настройка Telegram Bot

После развертывания:

1. В BotFather используйте команду `/setwebapp`
2. Выберите вашего бота
3. Введите URL: `https://<ваш-username>.github.io/<название-репозитория>/`

## Важные замечания

- GitHub Pages использует HTTPS, что необходимо для Telegram Mini Apps
- При первом запуске может потребоваться до 5 минут для доступности сайта
- Если сайт не открывается, проверьте, что в настройках репозитория GitHub Pages включена
- Убедитесь, что файл `vite.config.ts` содержит `base: './'` для корректной работы с подпапками GitHub Pages

## Возможные проблемы и решения

### Проблема: Сайт не загружается или показывает только "test"
**Решение:** Убедитесь, что в `vite.config.ts` установлено `base: './'` и сборка завершена успешно

### Проблема: Изображения или стили не загружаются
**Решение:** Это может быть связано с неправильным базовым путем, проверьте настройки сборки

### Проблема: Приложение не работает в Telegram
**Решение:** Убедитесь, что сайт доступен по HTTPS и что в BotFather правильно указан URL
